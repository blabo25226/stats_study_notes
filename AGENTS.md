# AI Assistant Guidelines (AGENTS.md)

## 1. プロジェクトの目的とAIの役割
- **目的**: 2026年11月の統計検定1級合格、および将来の大学院入試に向けた数理統計学の学習ノート構築。
- **AIの役割**: ユーザーの学習アシスタント。数式の展開、証明の補足、R/Pythonを用いたグラフ作成、マークダウンのフォーマット整備、Gitコミットメッセージの作成を支援する。厳密な数学的正確性を最優先すること。***特に私が手書きした数式の写真を.md内にtexに変換すること。***

## 2. ディレクトリとファイル作成ルール
- **ファイル分割の例外**: 
  - `現代数理統計学の基礎` と `統計数理過去問` `統計応用過去問` に関しては、　**必ず1問題につき1ファイル（.md）**　を作成すること。
  - フォルダとファイル名の先頭には連番を付与しない（例: `問1.md`）。
- **フロントマター（YAML）**:
  - obsidian用に全ての `.md` ファイルの先頭にYAMLヘッダーを追加し、分類（タグ）や理解度などを記述すること。タグは以下の通り。追加予定有り。
    - `test-category`: 試験の分類（院試, 統計数理, 統計応用）
      - 院試: 統計検定１級範囲外の院試用の学習内容の場合と、院試の過去問の場合付与する。
      - 統計数理: 統計数理用の対策の学習内容の場合と、統計数理の過去問の場合付与する。現代数理統計学の基礎の内容は原則これである。
      - 統計応用: 統計応用の対策の学習内容の場合と、統計応用の過去問の場合付与する。主に生物統計、医療統計学である。
    - `stats-category`: 内容に分類（例: 集合, 確率,,,）現代数理統計学の基礎については同じ章では同じものを分類する。(例 第1章:集合,第2章:確率など)
    - `difficulty`: 難易度（易, 中, 難）ここはとりあえずAIが適切なものを選んで。
    - `status`: 理解度（未着手, 理解不能, 学習中, 一度解けた,完了, 要復習）現代数理統計学の基礎については回答したものについては"一度解けた"を付与すること。完了は原則AIは付与せず、私が決めて編集する。
  - フォーマットは `docs/format/` の指定に従うこと。必要に応じて拡張を提案してほしい。
- **画像の保存**:
  - AIがPython/Rスクリプトでグラフ（確率密度関数など）を生成したい場合、スクリプトは `src/create_graphs/` に、生成された画像は `data/` に保存すること。

## 3. フォーマットとObsidian記法ルール
- **数式 (LaTeX)**:
  - ブロック数式は `$$` で囲み、インライン数式は `$` で囲むこと（Obsidianで正しくレンダリングさせるため）。
  - 数式内での日本語の使用は避け、必要な場合は `\text{}` を用いること。
- **リンク**:
  - 他のノートへの参照は、Obsidianの内部リンク記法 `[[ファイル名]]` を使用すること。

## 4. 著作権とコンプライアンス（厳守事項）
- **問題文の取り扱い**:
  - 教科書（現代数理統計学の基礎など）や過去問の**問題文をそのままMarkdown内に生成・書き写すことは絶対に行わない**こと（著作権保護のため）。ただし、これから作るファイルが.gitignoreに入る場合はその限りではなく、むしろそのまま引用することを推奨する。
  - ノートは「前提条件の数式」や「解答の展開」から直接書き始めること。

## 5. ワークフロー（日報と履歴）
- **作業過多の場合**: もし私からかAGENTS.mdの指示においてその作業が大きすぎて、1作業が当たりの最大トークン量に触れそうな場合は作業を分割し、その都度、私のgo!をもらってください。このとき、分割する割合はなるべく分割数が少なくすること。
- **Daily Report**:
  - 作業終了時、またはユーザーからの指示があった場合、`daily_report.md` にその日の学習内容、作成・更新したファイル、躓いたポイントを追記すること。
